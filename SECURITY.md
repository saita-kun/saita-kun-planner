# Security Policy

サイタくん（saita-kun-planner）は、Claude Code 契約者が自分のリポジトリで補助金申請用の事業計画書の叩き台を作るための OSS template repo です。セキュリティ上の問題を見つけた場合は、公開 issue ではなく、下記の私的報告経路から連絡してください。

## 報告経路

1. 主経路: GitHub の **Private Vulnerability Reporting** を使ってください。GitHub 上のこのリポジトリで `Security` タブを開き、`Report a vulnerability` から報告します。
2. 副経路: GitHub の私的報告が使えない場合は、`info@subsidy-support.tech` へメールしてください。

公開 issue、公開 Pull Request、SNS、チャットなどに脆弱性の詳細や再現手順を投稿しないでください。

## 対象範囲

対象は、このリポジトリが配布する中核ハーネスです。

- `.claude/commands/` の slash command プロンプト
- `tools/` 配下の検証・更新スクリプト
- `schemas/`、`specs/`、`templates/` の構造化ファイル
- `docs/`、`README.md`、`TERMS.md` など、利用者の安全な運用に影響する公開ドキュメント
- `core-manifest.json` と update-core 系の対象範囲定義

## 対象外

次の内容は、このリポジトリのセキュリティ脆弱性報告の対象外です。

- 利用者が自分の repo で生成した申請書、事業計画書、レビュー結果、提出前チェックリスト
- 利用者の `input/`、`knowledge/`、`my-*` ファイル、会社情報、見積、売上、個人情報などの実データ
- 利用者の端末、GitHub アカウント、Claude Code アカウント、外部 SaaS の設定不備
- 補助金の採否、審査判断、行政書士法その他の専門的な法令判断

利用者の実データが関係する相談は、公開 issue に貼らず、手元の private repo から必要最小限の再現条件だけを切り出してください。

## 報告に含めてほしい情報

- 影響を受けるファイル、コマンド、スクリプト
- 再現手順と期待される挙動
- 影響範囲の見立て
- 利用した環境（OS、shell、Python 版など。個人情報は不要）
- 可能であれば、修正案または緩和策

顧客名、取引先名、売上、投資額、補助金の申請内容、個人情報などは含めないでください。

## 応答目安

- 受領確認: 7日以内を目安に返信します。
- 初期評価: 14日以内を目安に、対象範囲かどうか、深刻度、次の対応方針を共有します。
- 修正・公開: 影響範囲と修正難度に応じて、できるだけ早く対応します。

この目安は保証ではありませんが、報告を放置しないための運用基準です。

## 公開までの非開示のお願い

修正または緩和策が公開されるまで、脆弱性の詳細を第三者へ公開しないでください。利用者の private repo や補助金申請データに影響しうる場合、先に安全な移行・削除・更新手順を用意する必要があります。

このリポジトリは補助金申請者の自己作成を支援する道具であり、作成者は顧客本人です。セキュリティ対応でも、申請内容そのものの代筆、申請代行、代理提出は扱いません。
