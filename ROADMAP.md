# ROADMAP

このロードマップは、saita-kun-planner を補助金申請者が自分で使える公共財として育てるための将来方向です。実装時期や提供順を確約するものではなく、公式募集要項を一次情報とし、作成者は顧客本人であるという現行方針を前提にしています。

## 採択後モジュール

現行スコープは、補助金申請用の事業計画書を「提出できる状態まで」整えることです。将来は採択後に必要になる交付申請、実績報告、証憑整理、期限管理までを扱えるモジュールを検討します。採択後工程でも、AI は進捗整理とチェック補助に徹し、顧客本人が公式資料と事務局案内を確認する前提を崩しません。

## spec レジストリの公共財化

構造化した補助金 spec を、AI が読める補助金フィードの生データとして無料公開する方向を目指します。募集要項の文章をそのまま代替するものではなく、原文確認へ戻れる clause、confirmation、鮮度情報を重視します。データライセンスの明示は今後の検討対象であり、再利用しやすさと制度情報の正確性を両立させます。

配布の形は、この repo への全量同梱ではなく、補助金パック単位で取得できる公開データリポジトリ方式を想定しています。取得したパックの完全性は sha256 台帳で機械検証でき、この repo 側の導入手順から同梱パックと同じ扱いで使える形を目指します。設計の方向は [docs/design/repo-structure.md](docs/design/repo-structure.md) に記録しています。

なお、レジストリは取得を便利にする利便レイヤであり、このキットの生命線ではありません。上流やレジストリが止まっても、各利用者は公式の募集要領から入口B（`/ingest-guidelines` → `/confirm-spec`）で spec を自作して自走を続けられます（設計不変条件: [DR-004](docs/design/decisions/dr-004-upstream-independent-self-run.md)）。

## 補助金パックの拡充

補助金パックの品揃えは、持続化補助金から始め、将来はものづくり補助金、IT導入補助金、省力化投資系の補助金へ広げる方針です。ただし、この repo に同梱するパックは代表的・利用の多い少数に絞り、品揃えの拡充分は公開データリポジトリ（spec レジストリ）経由の配布を想定しています。各パックでは、補助率や補助上限のような数値を固定知識として断定せず、公募回ごとの募集要項と confirmation に紐づけます。利用者が自社の投資計画に合う制度を選びやすいよう、要件、提出物、審査観点を比較しやすくします。

## 公募要領改訂の監視パイプライン

補助金 spec は、作った瞬間ではなく、公募回ごとの改訂に追随できて初めて安全に使えます。将来は公募要領、様式、FAQ、事務局案内の改訂を検知し、変更点を spec と confirmation に反映する監視パイプラインを整えます。鮮度を失ったデータは無料でも有害になり得るため、古い spec を見分ける仕組みを中核に置きます。

## 政策提言

任意 result-report などから得られる匿名集計を使い、どの要件や様式が申請者の負担になっているかを公開できる状態を目指します。個社情報や申請本文を出さず、集計として安全に扱える範囲に限定します。そのうえで、AI と構造化データによって申請プロセスをどこまで簡素化できるかを示し、制度側への改善提言につなげます。
