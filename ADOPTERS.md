## 掲載方法

掲載の申告は、canonical repo の Issue フォームからのみ受け付けます:

**<https://github.com/saita-kun/saita-kun-planner/issues/new?template=adopter-entry.yml>**

- この repo（公開版）への直接 Pull Request は受け付けていません。公開 repo は内部の正本から一括生成されるため、直接 PR の変更は次回の更新で消えてしまいます。
- 投稿した GitHub アカウントは Issue 上で公開されます。掲載時の表示名にはアカウント名と別の名称を指定できます。
- 掲載の削除依頼も同じ Issue フォームの導線（canonical repo の Issue）で受け付けます。ただし、Issue を閉じたり掲載を削除しても、GitHub の公開履歴・通知・複製からは完全には消えません。投稿前にこの点を了解してください。

## 掲載規定

- 掲載を申告できるのは、**本人または掲載の権限を持つ者のみ**です。支援者が顧客を無断で掲載することはできません（なお、支援者が掲載できることと、顧客の申請書を代理で作成してよいことは別の問題です。後者は本キットの法務スコープ — 作成主体は申請者本人 — に従います）。
- **書いてはいけないもの**: 制度の正式名称・公募回・年度・地域（補助金カテゴリは粗い分類のみ）/ 申請中の情報 / 申請本文・事業計画の具体的内容 / 売上等の具体的数値 / 顧客・第三者の特定につながる情報 / 個人情報。
- **採否・採択率は記載できません**。掲載は補助金の結果とは無関係です。
- 「公認」「提携」「認定」を示唆する表現は使えません。
- 掲載は自己申告であり、運営は内容を検証しません。**掲載は推薦・認定・成果の保証ではありません。**
- 運営は、規定違反・疑義のあるエントリを予告なく削除できます。
