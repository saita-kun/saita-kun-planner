> **発効日: 2026-07-05**

> 配布ライセンス（コード・テンプレの使用許諾）は `LICENSE` を参照。本 TERMS はサービス利用条件・データ条項を定める別レイヤ。

## 1. 本キットの位置づけ

本キット（saita-kun-planner）は、Claude Code を使えるクライアントが、補助金申請用の事業計画書の叩き台を**顧客本人が作成**するための情報補助ツールです。処理はクライアント側で実行され、事業計画書・申請本文・`input/` 配下の生データには提供側が構造上一切触れません。行政書士法に関する基本方針・スコープは `docs/法務とスコープ.md` を正本とし、本規約はこれを前提とします（作成者は顧客本人、AI は補助のみ、申請代行・代理提出はしない、数値は推測しない、募集要項優先、`[要確認]` 運用）。

## 2. result-report の任意提出

旧 collaborator 常時招待モデルは撤回します。利用者が提供側の GitHub アカウント／org を private repo の collaborator として受け入れることは、本キットの利用条件ではありません。

利用者は、採否判明後などに result-report を任意で提出できます。result-report は `docs/governance/data-charter.md` §4 の allowlist に限る狭い export であり、申請本文・事業計画書本文・会社名・代表者名・住所詳細・電話・メール・URL・法人番号・口座/税務情報・添付資料・prompt / AI 応答の生テキスト・生ログ・Git remote・端末ユーザー名・`input/` 配下の生データは送信しません。

result-report で扱う項目は、構造化メタデータと採否結果に限定します。具体的には、メタ情報（`report_id`, `schema_version`, `core_version`, `submitted_at_month`）、補助金情報、採否結果、事業者プロファイルの band/enum、eligibility の boolean/enum フラグ、検証情報（validate pass/fail, missing field 数, schema errors）です。

result-report は任意提出です。非提出でも、中核ハーネス（ローカル実行の slash command、スキーマ、検証ゲート、マニュアル等）の全機能は利用できます。提出した場合、その提出実績には公式フィード接続・update-core・最新 spec 受領などの継続的な公式サービスの付加価値を紐づけることがあります。これは非提出者を中核機能から除外する条件ではありません。B2B/有料契約の顧客は、result-report を提出しなくても同等機能を利用できる契約上の逃げ道を持ちます。

提出時は、送信前に JSON プレビューを表示し、`--consent result-report-vYYYY-MM-DD` または対話確認により consent version を記録します。初回実行時は FTUX バナーで収集内容・収集者・無効化方法・詳細 URL を告知し、告知前には一切送信しません。ドライランでは送信せず内容だけを表示します。

提出後の削除・撤回のため、提出時に削除用 ID / receipt を発行します。削除請求があった場合、raw store から該当レコードと紐付け情報をパージします。既に公開・集計済みのスナップショットは遡及変更しませんが、次回再集計から除外します。

## 3. データ利用条件

result-report の用途は、当面 **キット品質の改善（prompt・テンプレート・要件マッピング・ドキュメントの改善）** と、将来のマッチング品質改善に限定します。提供側は result-report の個票を第三者・スポンサー・B2B 顧客に渡しません。提供側は収集データを **第三者に外販しません**。

対外公開は `docs/governance/data-charter.md` §6 に従い、秘匿処理済みの集計のみとします。row-level 公開は禁止します。スポンサー資金は推薦順位・診断結果・補助金フィード採用・重み付け・検証ゲートに影響しません。

詳細は `docs/data-policy.md`（本条の平易版）を参照。

## 4. 二層 GTM

| 層 | 契約・提供単位 | result-report | 課金・特典 |
|----|----------------|---------------|------------|
| 事業者直販 | 利用者（事業者）単位 | **任意提出**（第2・3条）。非提出でも中核機能は利用可能 | キットは安価/無料。任意提出者には公式サービスの付加価値を紐づけることがある |
| 支援者/B2B 契約 | **支援者単位・クライアント数無制限** 等 | **データ提供なしで同等機能を利用可能** | **ライセンス料・契約料** |

- 事業者直販・支援者/B2B 契約のいずれでも、collaborator 受入れを利用条件にしません。
- 支援者は契約範囲内で複数のクライアント案件に本キットを利用できます。再配布・転売はできません（クライアントへの導入支援としての利用に限る）。
- 士業・支援者が顧客本人/法人を代表して result-report を提出する場合、本人/法人を代表して consent する権限を取得済みである旨の表明を求めます。
- 行政書士を公式座組に組み込むことは当面行いません。士業・支援者の利用は、`docs/法務とスコープ.md` の行政書士法スコープと本規約の非接触原則に従うものとします。
- 層別の詳細は `docs/licensing-tiers.md` を参照。

## 5. 行政書士法スコープ（再掲・正本は法務とスコープ.md）

- 提供側は、`docs/governance/data-charter.md` §0 の非接触原則に従い、申請書類・事業計画書本文・`input/` 配下の生データに一切触れません（処理はクライアント側 Claude Code で実行）。
- 作成者は顧客本人であり、提供側・AI は申請代行・代理提出・本人に代わる完成判断・官公署への提出代行を行いません。

## 6. プロンプト改変

利用者がキット内のプロンプトを書き換えることは妨げません。改変による結果は利用者の責任とします。

## 7. 改定

本規約は 2026-07-05 に発効しました。実質的な改定を行う場合は、変更内容と新しい発効日を明記します。
