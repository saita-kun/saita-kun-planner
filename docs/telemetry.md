# データ提供について（result-report 開示ドキュメント）

> **発効日: 2026-07-05**
> 本書は `docs/governance/data-charter.md` §4-6 を親規範とする開示ドキュメントです（`derived_from: docs/governance/data-charter.md`）。正式な条件は `TERMS.md`、平易版は `docs/data-policy.md` を参照。
> Homebrew / Next.js / CNCF などの OSS が採る「何を・なぜ・どこへ・保持期間・無効化方法を全開示する」水準に合わせています。

## 大前提: 提供側はあなたの申請内容に触れません

このハーネスの処理はすべて**あなたの環境（あなたが持ち込む AI）で実行**されます。事業計画書・申請本文・`input/` 配下の生データは、提供側に構造上一切送られません（`input/` は `.gitignore` で隔離）。提供側に届きうるのは、下記の **result-report** を**あなたが任意で提出した場合のそれだけ**です。

## result-report は任意です

- 提出は**任意**です。**非提出でも、ハーネスのローカル機能（slash command・スキーマ・検証ゲート・マニュアル）はすべて使えます。**
- 提出した場合、提出実績には公式サービスの付加価値（公式フィード接続・update-core・最新 spec 受領）を紐づけることがあります。これは提出者への付加価値であり、非提出者を中核機能から締め出す条件ではありません。
- B2B・有料契約の場合は、result-report を提出しなくても同等機能を利用できます。

## 何を集めるか（allowlist に相当する収集項目）

result-report は**構造化メタデータと採否結果のみ**を扱います。

- メタ: `report_id`, `schema_version`, `core_version`, `submitted_at_month`
- 補助金: `subsidy_id`, `subsidy_name`, `round`, `jurisdiction_level`
- 結果: `result_status`（採択/不採択/取下げ等）, `result_month`, `requested_amount_band`（金額レンジ）, `awarded_amount_band`（金額レンジ）
- 事業者プロファイル（すべて区分＝band/enum、生値ではない）: 業種（大/中分類）, 地域（都道府県/地方）, 従業員数階級, 売上階級, 業歴階級, 法人格区分
- 適格条件: eligibility の true/false/unknown フラグ
- 検証情報: validate の pass/fail, 未入力フィールド数, スキーマエラー

## 何を集めないか（denylist）

次のものは **送りません**。

- 申請本文・事業計画書本文
- 会社名・代表者名・住所詳細・電話・メール・URL・法人番号
- 口座・税務情報・添付資料
- プロンプト／AI 応答の生テキスト・生ログ
- Git remote 情報・端末ユーザー名
- 自由記述欄
- `input/` 配下の生データ全般

## 同意の取り方

- 提出コマンドは**送信前に「何を送るか」の JSON プレビュー**を表示します。
- `--consent result-report-vYYYY-MM-DD` または対話確認で **consent version を記録**します。
- 初回実行時に、収集内容・収集者・無効化方法・本書 URL を告知するバナーを表示し、**告知前は一切送信しません**。
- **ドライラン**（送信せず内容だけ表示）を提供します。技術的な強制はしません（OSS なので、送るか送らないかはあなたの操作次第です）。
- 士業・支援者があなたに代わって提出する場合は、あなた（本人/法人）を代表して同意する権限を取得済みであることの表明を求めます。

## 削除・撤回

- 提出時に**削除用 ID / receipt** を発行します。削除請求があれば、raw store の該当レコードと紐付け情報を消去します。
- ただし、すでに公開・集計済みのスナップショットは遡及変更しません（次回の再集計から除外します）。完全に匿名化・集計化されたデータは個人データにあたらないため、個票としての削除対象になりません。

## 用途・保管・公開

- 用途は当面「キット品質の改善」と将来の「マッチング品質改善」に限定します。
- **個票は内部限定**（モデル学習等）。**対外公開は秘匿処理済みの集計のみ**（個社が特定されないよう匿名化。詳細は `docs/governance/data-charter.md` §6）。
- **個票を第三者・スポンサー・B2B 顧客に渡しません。第三者への外販もしません。**

## 問い合わせ

収集内容・削除・撤回の窓口は本リポジトリの運営者情報（README / `docs/`）を参照してください。
