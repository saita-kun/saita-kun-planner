# データの扱いについて（やさしい版）

> **発効日: 2026-07-05**
> 本書は `TERMS.md` 第2〜3条の平易な説明です。正式な条件は `TERMS.md`、親規範は `docs/governance/data-charter.md` です（`derived_from: docs/governance/data-charter.md`）。

## 何のためにデータを集めるのか

このキットをもっと使いやすく、補助金の条件に合う叩き台を作りやすいツールに改善し続けるためです。将来は、どの補助金がどんな事業者に合いやすいかを判断するマッチング品質の改善にも使うことがあります。

## 何を集めるか

集めるのは、あなたが任意で提出した result-report だけです。内容は `docs/governance/data-charter.md` §4 の allowlist に限ります。

- 採否結果（採択/不採択/取下げなど）、結果が出た月、申請額・採択額のレンジ
- 補助金名、回、国/都道府県/市区町村などの制度区分
- 粗い事業者属性（地域は都道府県または地方ブロック、業種は大分類/中分類、金額・従業員数・売上・事業年数はレンジ、法人種別）
- 適格条件に当てはまったかどうかの yes/no や選択式の結果
- 検証結果（validate の pass/fail、足りない項目数、schema errors）
- report ID、schema/core version、提出月などの管理用メタデータ

## 何を集めないか

以下は集めません。提供側には届きません。

- 申請本文、事業計画書本文、叩き台本文
- 会社名、代表者名、住所、電話、メール、URL、法人番号
- 売上や申請額などの具体的な生の数値（送る場合もレンジだけ）
- 口座/税務情報、添付資料、`input/` 配下のファイル
- prompt、AI 応答、対話ログ、生ログ
- Git remote、端末ユーザー名

`input/` は最初から git 管理外（`.gitignore`）で、あなたの repo の中に留まります。

## 出さなくても使えるか

使えます。result-report は任意提出です。非提出でも、ローカルで動く中核ハーネスの全機能は使えます。

提出した人には、公式フィード接続、update-core、最新 spec 受領などの公式サービスの付加価値を紐づけることがあります。これは非提出者を中核機能から除外する条件ではありません。B2B/有料契約では、result-report を出さずに同等機能を使える道を残します。

## どう守るか

- 送信前に JSON プレビューを表示し、consent version を記録します。
- 初回は何を集めるかを告知し、告知前には送信しません。
- ドライランでは送信せず、送られる内容だけ確認できます。
- 個別の result-report は第三者・スポンサーに渡しません。
- 公開する場合は、個社がわからないように秘匿処理した集計だけにします。
- 用途はキット改善と将来のマッチング品質改善に限ります。**第三者に売りません（外販しません）**。

## 削除したいとき

提出時に削除用 ID / receipt を発行します。削除したい場合は、その ID / receipt を添えて、提供側の削除請求窓口（README または配布時に明示する連絡先）へ連絡してください。

削除請求があった場合、元データ（raw store）から該当レコードと紐付け情報を削除します。すでに公開・集計済みのスナップショットは戻って直しませんが、次回の再集計から除外します。
