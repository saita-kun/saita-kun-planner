# 法務とスコープ

この文書は、saita-kun-planner を使う範囲を明確にするための注意書きです。このキットは、Claude Code 契約者である補助金申請者が、自分の資料と公式の募集要項を確認しながら、補助金申請用の事業計画書の叩き台を作るための情報補助ツールです。

## 基本方針

作成者は顧客本人です。AI は補助・壁打ち・整理役であり、顧客本人に代わって申請書を完成させたり、提出可否を判断したりする立場ではありません。

このキットでできることは、会社情報、課題、投資計画、募集要項の要件、数値根拠、未確認事項を整理し、顧客本人が確認・修正するための叩き台やチェックリストを作ることです。最終的な内容確認、修正、提出判断、提出作業は顧客本人が行います。

## しないこと

このキットと AI は、行政書士法に抵触する申請代行、代理提出、本人に代わる完成判断、官公署への提出代行を行いません。顧客本人の確認を経ずに「この内容で提出できる」「採択される」「要件を満たす」と確定することもしません。

また、税理士法や行政書士法などの専門業務に踏み込む判断は行いません。税務処理、会計処理、許認可、法令適合性、申請資格の確定、対象経費の可否、提出方法の最終判断など、専門的な判断が必要な場合は、行政書士、税理士、商工会議所、商工会、よろず支援拠点、補助金事務局など、適切な専門家または公的支援機関へ確認してください。

## 数値・要件の扱い

数値は推測しません。売上、従業員数、投資額、見積額、補助率、補助上限、採択率、効果見込み、KPI、文字数、提出期限、添付資料、対象経費は、顧客の資料または公式の募集要項で確認できる場合だけ断定してください。

補助金の要件・数値・様式は、公式の募集要項が正です。このリポジトリの説明、テンプレート、AI の出力と募集要項が食い違う場合は、必ず募集要項を優先してください。出典不明の事実、根拠が不足している主張、顧客本人の確認が必要な情報には `[要確認]` を付けて残します。

## AI 出力の位置づけ

`/intake`、`/subsidy-fit`、`/draft-section`、`/review`、`/finalize` の出力は、提出文書そのものではありません。顧客本人が自社資料、見積書、募集要項、公式 FAQ、指定様式と照合し、必要な修正を行うための作業メモまたは叩き台です。

AI が出した文章を使う場合でも、顧客本人が内容を読み、事実関係、数値、根拠、表現、提出先の様式との一致を確認してください。根拠が弱い主張、強すぎる表現、確認できない効果見込みは、提出前に修正または削除してください。

## 迷ったとき

申請者本人だけで判断しづらい場合は、AI に判断を委ねず、公式窓口や専門家に確認してください。特に、対象者・対象経費・補助率・補助上限・提出期限・添付資料・税務処理・許認可・電子申請の操作は、制度ごとの公式情報を最優先に扱います。

このキットは、顧客本人が作成主体として情報を整理し、補助金審査の観点に沿った事業計画書の叩き台を作るための環境です。申請代行や代理提出を目的にしたものではありません。
