# 補助金の選び方

この文書は、Claude Code 契約者が saita-kun-planner を使って事業計画書の叩き台を作る前に、「どの補助金を対象にするか」を整理するための地図です。一次情報は必ず公式の募集要項、公募ページ、要領、様式、FAQ、審査項目を優先してください。本書は探索の地図であり、特定の補助金への適合、採択可能性、補助額を断定するものではありません。

作成者は顧客本人です。AI は情報整理、比較、質問づくりを補助しますが、申請書の作成代行、代理提出、本人に代わる完成判断は行いません。制度名、対象者、対象経費、補助率、補助上限、締切、必要書類、文字数など、公式資料で確認できない項目には `[要確認]` を付けてください。

## 使うタイミング

次の状態なら、この文書を先に使ってください。

- 対象にしたい補助金がまだ決まっていない。
- 候補はあるが、自社の事業や投資計画に合うか判断できていない。
- 補助金名だけ知っていて、公式の募集要項をまだ読めていない。
- 締切までに準備できるか、逆算できていない。

対象補助金と公式資料が決まったら、同梱の補助金パックを使う場合は `/select-subsidy`、自分の募集要項から作る場合は `/ingest-guidelines` → `/confirm-spec` で confirmed spec を先に確定します。confirmed spec になったら `/build-pack` で書き方メモを作ると、後続の `/draft-section` と `/review` で制度別の観点を参照しやすくなります。その後 `/intake` で spec が求める会社情報を整理し、`/subsidy-fit` で会社プロフィールと募集要項を照合します。

## 探し方

補助金を探すときは、最初からブログやまとめ記事だけで判断しないでください。入口として参考情報を読むことはできますが、要件と数値は必ず公式情報源で確認します。

### 公式情報源として確認する場所

- J-Net21 など、公的機関が運営する中小企業向け施策検索ページ
- Jグランツなど、補助金や助成金の申請・検索に使われる公式系ポータル
- 国の省庁、中小企業庁、地方自治体、外郭団体、補助金事務局の公式ページ
- 商工会議所、商工会、産業振興センターなど、地域の公的支援機関の案内
- 募集要項、公募要領、交付要綱、申請様式、記入例、FAQ、審査項目

### 対象の補助金がまだ決まっていない場合（Jグランツで探す）

国の補助金の電子申請システム「Jグランツ」の Web 検索（<https://www.jgrants-portal.go.jp/>）で、公募中の補助金を業種や地域から探せます。まずはこの通常の Web 検索で候補を見つけるのが確実です。

AI から補助金を検索したい場合は、[`jgrants-mcp-server`](https://github.com/digital-go-jp/jgrants-mcp-server) を利用する方法もあります。配布元 README では、本実装を、技術検証を目的として公開されているサンプルコードであり、安定性・継続的な保守・検索性は保証されない、としています（出典: <https://github.com/digital-go-jp/jgrants-mcp-server>）。利用は Jグランツ API の利用規約（出典表示を含む）に従ってください。なお、本キットとデジタル庁・Jグランツの間に提携・公認の関係はありません。

どの経路で見つけた場合も、制度事実（対象者、補助率、補助上限、締切、様式）は必ず公式の公募要領で再確認してください。入口B `/ingest-guidelines` に投入するのは、その公式公募要領そのものです。

候補を見つけたら、少なくとも次の情報を `input/` にメモしてください。確認できない項目は空欄にせず `[要確認]` とします。

| 項目 | 確認する内容 |
| --- | --- |
| 補助金名 | 制度名、公募回、年度、事務局名 |
| 公式資料 | 募集要項、様式、FAQ、審査項目の所在 |
| 対象者 | 法人形態、所在地、業種、事業規模、創業時期など |
| 対象事業 | どのような取り組みが対象か |
| 対象経費 | 設備、システム、外注、広告、開発などの区分 |
| 補助率・補助上限 | 公式資料に書かれた数値。未確認なら `[要確認]` |
| 締切 | 申請締切、事業実施期間、契約・支払の制限 |
| 添付資料 | 見積書、決算書、登記、認定、宣誓、計画書など |
| 審査項目 | 課題、実現性、効果、政策目的、加点項目など |

## 自社との適合を見極める

補助金を選ぶときは、「もらえそうか」より先に「募集要項の対象に入るか」「自社の投資計画と審査観点がつながるか」を確認してください。次の順で見ると、後の `/subsidy-fit` に接続しやすくなります。

### 1. 除外条件に当たらないか

対象外業種、対象外事業、対象外経費、重複申請、事業開始時期、契約時期、支払時期、所在地、法人形態などを確認します。ひとつでも該当可能性がある場合は、申請前の重要論点として `[要確認]` を付け、公式資料や支援機関で確認してください。

### 2. 必須要件を満たせるか

対象者要件、対象事業要件、対象経費要件、提出書類、様式、文字数、期限、事前登録、認定、宣誓などを確認します。必要な資料を締切までに用意できない場合、その補助金は今回の計画に合わない可能性があります。

### 3. 審査で説明できる強みがあるか

補助金の政策目的と、自社の課題、投資内容、実施体制、効果がつながるかを確認します。たとえば、生産性向上、販路開拓、省力化、地域波及、賃上げ、雇用、デジタル化などが審査項目にある場合、自社の実情報で説明できるかを見ます。根拠資料がない効果見込みは断定せず `[要確認]` とします。

## 締切から逆算する

候補補助金が見つかったら、締切だけでなく、申請前後の制約を逆算してください。補助金によっては、採択前の契約、発注、支払が対象外になる場合があります。必ず募集要項の定義を確認してください。

逆算では、次の順に確認します。

1. 申請締切と提出方法
2. 事前登録、認定、宣言、支援機関確認などの有無
3. 見積書、仕様書、決算書、登記、納税、認定書類などの準備期間
4. 事業計画書の文字数、様式、入力欄、添付ファイル形式
5. 採択後の契約、発注、納品、支払、実績報告の期限
6. 社内で顧客本人が確認・修正する時間

締切が近い場合は、文章作成より先に「必要資料が揃うか」「対象経費の時期が合うか」「顧客本人が確認できる時間があるか」を見てください。ここを確認しないまま事業計画書を書き始めると、後から対象外や準備不足が見つかることがあります。

## よくあるミスマッチ

次の状態は、候補として残す前に注意してください。

- 自社の所在地、業種、規模、法人形態が対象者要件に合っているか未確認。
- 投資したい内容が、募集要項の対象事業や対象経費に入るか未確認。
- 補助率、補助上限、自己負担額を公式資料ではなく一般論で見積もっている。
- すでに契約、発注、支払を済ませた経費を対象にしようとしている。
- 審査項目に対して、課題、解決策、実施体制、効果の説明がつながらない。
- 加点項目を狙っているが、認定書、宣言、計画書、根拠資料がない。
- 締切までに見積書、添付資料、社内確認、顧客本人の修正時間を確保できない。
- 第三者の記事の説明と公式募集要項の記載が食い違っている。

ミスマッチがある場合でも、すぐに不採択と断定しないでください。まず `[要確認]` として、募集要項の該当箇所、公式 FAQ、事務局、支援機関で確認すべき論点に分けます。

## Claude Code での整理方法

候補が複数ある場合は、Claude Code に次のような整理を依頼できます。ただし、公式資料が手元にない状態で要件や数値を推測させないでください。

```text
候補補助金を比較したいです。
以下の公式募集要項メモをもとに、除外条件、必須要件、審査観点、締切、準備資料を表にしてください。
確認できない項目は [要確認] としてください。
採択可能性や補助額は断定しないでください。
```

比較表は、次の形式にすると `/select-subsidy` または `/ingest-guidelines` へ進む判断をしやすくなります。

| 候補 | 公式資料 | 自社との適合 | 締切逆算 | 主な不足準備 | 次の判断 |
| --- | --- | --- | --- | --- | --- |
| 候補A | [要確認] | [要確認] | [要確認] | [要確認] | `/select-subsidy` または `/ingest-guidelines` に進むか確認 |
| 候補B | [要確認] | [要確認] | [要確認] | [要確認] | 保留または除外理由を確認 |

## 次に進む判断

次の条件が揃った候補から、同梱 spec が対象回と一致するなら `/select-subsidy`、公式募集要項から自分で spec を作るなら `/ingest-guidelines` に進んでください。入口Bでは `/ingest-guidelines` の後に `/confirm-spec` で募集要項との突合確認を行い、必要に応じて `/build-pack` で書き方メモを作ります。

- 公式の募集要項、様式、FAQ、審査項目を確認できる。
- 自社が対象者要件に入りそうだが、断定前に照合したい。
- 投資計画が対象事業・対象経費に入りそうだが、根拠確認が必要。
- 締切までに必要資料を準備できる見込みがある。
- 事業計画書で説明する課題、解決策、実現性、効果の材料がある。

confirmed spec ができたら、`/build-pack` で補助金パックを整えてから、`/intake` でその spec が求める会社プロフィールを整理し、`/subsidy-fit` で公式募集要項との照合に進みます。最後まで、数値・要件は公式資料を優先し、出典不明の情報には `[要確認]` を残してください。
