# DR-006: 禁止表現検査は自己完結・文単位判定

> 種別: 設計不変条件 ／ 状態: 有効 ／ 決定日: 2026-07-17 ／ 決定者: DRI ／ 出典: 内部決定記録（非公開）

## 決定

顧客向けテキストの禁止表現検査（成果保証系・全自動系・代行系）は、**repo 内で自己完結**し（外部リストへの実行時依存なし）、**文単位**で判定する: テキストを文（`。`・改行）単位に分割し、否定語を含む文は判定対象から除外した上で、残る文に肯定パターンを検査する。

## 理由

- 行単位判定は「自動で完成させるものではありません」等の既存法務定型文を誤検出し、逆に同一行の無関係な否定によって肯定文を見逃す。文単位でなければ精度が出ない。
- 検査リストを外部ファイル・外部 repo と同期にすると、fork 単体で検査が完結しなくなる（DR-004 の上流非依存に反する）。由来はコメントで記録しつつ固定版・独立運用とする。

## 制約（運用 AI が守ること）

- 走査範囲は export tree に含まれる顧客向けテキスト全体（除外パスは export 側と単一定義を共有し、二重定義しない）。
- 肯定パターン（成果保証: 「採択されます」「保証します」等／全自動: 「完全自動」「自動で申請」等／代行: 「作成代行」「代理提出」「成功報酬」等）を弱めるときは、negative fixture（既存法務定型文が FAIL しないこと）と adversarial fixture（肯定表現＋無関係な否定文の同居が FAIL すること）を必ず更新する。
- 検査を外部 URL やネットワーク取得に置き換えない。

## 違反例

- 既存の法務定型文が FAIL したので検査ごと削除する（正しい対処は文単位判定・否定語除外の修正）。
- 表現リストを別リポジトリから実行時に取得する構成に変える。
