# 03. このキットを自分のものにする

GitHub アカウントと Claude Code が用意できたら、このキットを**あなた専用の private repo**にして、手元で開きます。

## 手順

1. **テンプレートから自分の repo を作る**
   - このキットの GitHub ページで緑色の **「Use this template」**（このテンプレートを使う）ボタンを押します。
   - 「Create a new repository」を選びます。
   - repo 名を決めます（例：`my-subsidy-plan`）。
   - 公開範囲は必ず **Private（非公開）** を選びます。
   - 「Create repository」を押します。

2. **自分のパソコンに clone（ダウンロード）する**
   - 作成した repo のページで「Code」ボタン → URL をコピー。
   - ターミナルで `git --version` を実行し、git が入っているか確認します。
   - バージョン番号が出ない場合は、先に `02-claude-codeセットアップ.md` の「git のインストール」を確認してください。
   - ターミナルで作業したい場所に移動し、`git clone <コピーした URL>` を実行します。
   - できたフォルダに入ります（`cd <repo 名>`）。

3. **Claude Code で開く**
   - そのフォルダの中で `claude` を起動します。
   - これでキットの slash command（`/setup`, `/start`, `/intake` …）が使えます。

4. **`/setup` を実行する**
   - Claude Code で `/setup` と入力します。
   - 環境チェック、利用規約の同意、result-report 任意提出の扱いまで、Claude Code が一つずつ案内します。
   - 完了したら `/start` へ進みます。

5. **公募要領 PDF を置く**
   - 自分の募集要項から進める場合は、公式の公募要領 PDF または貼り付け本文を `input/guidelines/` に保存してから、`/ingest-guidelines` → `/confirm-spec` に進みます。
   - macOS では Finder で clone したフォルダを開き、`input` → `guidelines` の順に開いて、公募要領 PDF をそこへドラッグします。
   - Windows ではエクスプローラーで clone したフォルダを開き、`input\guidelines` を開いて、公募要領 PDF をそこへドラッグします。
   - `input/guidelines/` が見つからない場合は、clone したフォルダの直下を開いているか確認してください。

## 困ったら

- `docs/faq.md`（よくある質問・Claude Code 固有のつまずき）
- `docs/manual.md`（全体マニュアル）

## メモ

- `input/` フォルダにあなたの会社情報、募集要項 PDF、募集要項メモを置きます。ここは git 管理外なので、提供側にも公開リポにも出ません。
- 利用条件・データの扱いは `TERMS.md` / `docs/data-policy.md` を確認してください。
