# ライセンス二層（事業者直販 / 支援者・B2B）

> 正式な条件は `TERMS.md` 第4条。本書は二層の違いと運用の説明です。
> result-report とデータ利用の親規範は `docs/governance/data-charter.md`、平易な説明は `docs/data-policy.md` です。
> **発効日: 2026-07-05**

## 二層の比較

| 項目 | 事業者直販 | 支援者・B2B |
|------|-----------|-------------|
| 対象 | Claude Code を使える事業者本人 | 複数クライアントを支援するコンサル・士業・法人 |
| 契約・提供単位 | 利用者（事業者）単位 | **支援者単位・クライアント数無制限** 等 |
| result-report | **任意提出**。非提出でも中核機能は使える | **データ提供なしで同等機能を利用可能** |
| データ範囲 | `docs/governance/data-charter.md` §4 の allowlist に限る | 契約で別途定めない限り result-report なし |
| 課金・特典 | キットは安価/無料。任意提出者に公式フィード接続、update-core、最新 spec 受領などの付加価値を紐づけることがある | **ライセンス料・契約料** |

## なぜ二層を分けるか

- **事業者直販**：顧客本人が自分の repo でキットを使い、採否判明後などに result-report を任意提出できます。提出された report は、キット品質の改善と将来のマッチング品質改善に限って使います。非提出でも、ローカル実行の中核ハーネスは使えます。公式フィード接続や update-core などは、提出者への付加価値に連動することがあります。
- **支援者・B2B**：士業・コンサル・法人は、顧客の守秘義務や社内統制に配慮し、result-report を出さずに同等機能を使える契約上の逃げ道を持ちます。ここではデータ提供の代わりに、支援者単位のライセンス料・契約料を収益源にします。

いずれの層でも、提供側を利用者の private repo に共同作業者として招待することは利用条件ではありません。

## 支援者・B2B の運用

- 支援者は、契約範囲内で複数のクライアント案件にキットを利用できます。
- 各クライアント案件は、支援者または顧客本人がテンプレートから private repo として立ち上げます。
- 再配布・転売は不可です。クライアントへの導入支援としての利用に限ります。
- 士業・支援者が顧客本人/法人を代表して result-report を提出する場合は、本人/法人を代表して consent する権限を取得済みであることが前提です。
- 行政書士法スコープは `docs/法務とスコープ.md` を優先します。提供側・AI は申請代行、代理提出、本人に代わる完成判断を行いません。

## GTM 順序

現行方針では、直販と支援者・B2B を分けつつ、result-report はデータ憲章に沿った任意提出として扱います。公開・営業・継続特典の設計は、`TERMS.md` と `docs/governance/data-charter.md` に反する形へ広げません。
