# 補助金要件マッピング

このテンプレートは、`/subsidy-fit` で confirmed spec を読み取り、募集要項の要件が事業計画書のどのセクションに反映されるかを人間が確認するためのビューです。JSON spec が機械照合の正本であり、この Markdown は顧客本人が読みやすくするための補助表です。

要件・数値は募集要項が正です。確認できない項目、根拠が不足している項目、顧客本人の判断が必要な項目には `[要確認]` を付けてください。spec と公式募集要項が食い違う場合は、必ず公式募集要項を優先し、必要なら `/ingest-guidelines` で `input/spec/` と confirmation を作り直してください。

## 利用上の注意

- 作成者は顧客本人です。AI は募集要項と会社プロフィールの照合、不足点の指摘、事業計画書への反映先整理を補助するだけです。
- 申請書の作成代行、代理提出、本人に代わる完成判断は行いません。
- 募集要項、公募要領、様式、FAQ、審査項目など、公式情報源を一次情報として扱ってください。
- ブログ、まとめ記事、SNS、第三者の解説は参考情報にとどめ、要件や数値の根拠にしないでください。
- 補助率、補助上限、対象経費、締切、文字数、添付資料は推測せず、公式資料から確認してください。
- JSON spec は機械照合の正本です。この表を手で直しても `tools/check-spec.sh` や `tools/check-drafts.sh` の判定は変わりません。
- この表で見つけた不一致は、`input/subsidy-fit.md` の不足準備、または `input/spec/` の再確認事項に戻してください。

## spec から読む主なフィールド

| spec field | 人間が確認する内容 | 事業計画書への主な反映先 |
| --- | --- | --- |
| `subsidy_id` / `spec_version` / `status` | 対象補助金、公募回、confirmed かどうか | 全体の前提、提出前確認 |
| `source_documents[]` | 公式募集要項、公募要領、様式、FAQ などの根拠資料 | 参照資料、根拠管理 |
| `clauses[]` | 募集要項から抜き出した 1 clause = 1 論点の原文 | `/review` の `clause_id` / `quoted_text` 点検 |
| `eligibility.rules[]` | 除外要件、必須要件、加点要素 | 事業概要、現状課題、事業内容、実施体制、効果・KPI |
| `funding.base_award` / `funding.add_ons[]` / `funding.combinations[]` | 基本枠、上乗せ、組み合わせ、補助率・補助上限 | 資金計画、効果・KPI、提出前確認 |
| `schedule[]` | 申請締切、事業実施期間、実績報告など | スケジュール、添付準備、hard 締切 |
| `deliverables[]` | AI と作る成果物、人がやること、添付資料 | `/plan-deliverables`、`/draft-section`、`/finalize` |
| `deliverables[].sections[]` | 事業計画書の section、文字数、guidance、review_criteria | 各 draft の frontmatter と `## 叩き台` |
| `source_clauses` | 各要件・section がどの clause に基づくか | 根拠引用、誇張・捏造リスクの点検 |

## 適合度メモへ転記する観点

### 除外要件チェック

- 申請者が対象外に当たらないか
- 補助事業が対象外事業に当たらないか
- 経費、契約、発注、支払時期が対象外にならないか
- 他制度との重複や併用制限に抵触しないか
- 不明な項目は `[要確認]` として残す

### 必須要件チェック

- 対象者要件を満たす根拠
- 対象事業要件を満たす根拠
- 対象経費の範囲と見積根拠
- 必須の数値目標や計画条件
- 必須添付資料と準備状況

### 加点要素

- 募集要項に明記された加点項目
- 審査項目から見た強み
- 会社プロフィール側で裏付けられる実績
- 追加資料があれば強化できる論点
- 加点と断定できない項目の `[要確認]`

### 不足準備

- 公式資料で追加確認する箇所
- 顧客本人が判断する論点
- 見積書、仕様書、売上資料、作業時間記録などの根拠資料
- 支援機関や専門家に確認したほうがよい事項
- `/draft-section` に進む前に埋めたい情報

## 事業計画書セクション別の反映先

### 事業概要

対象者要件、会社の事業内容、商圏、顧客、既存実績を反映します。募集要項の対象者条件と矛盾しないか確認してください。

### 現状課題

補助事業が必要な理由を反映します。課題は一般論ではなく、売上、作業時間、顧客数、品質、設備制約など、顧客の実情報に基づけてください。

### 事業内容

対象事業要件、対象経費、導入する設備やシステム、実施内容を反映します。募集要項で対象外とされる事業や経費が混ざらないよう確認してください。

### 実施体制

担当者、外部協力先、実施能力、許認可、認定、支援機関の関与を反映します。体制が未確定の場合は `[要確認]` としてください。

### スケジュール

申請、採択、契約、発注、納品、支払、実績報告の時期を反映します。事業実施期間と経費対象期間を必ず募集要項で確認してください。

### 資金計画

対象経費、補助対象外経費、自己負担、資金調達、支払時期を反映します。補助率や補助上限は公式資料から確認し、推測しないでください。

### 効果・KPI

審査項目、加点項目、政策目的に対応する効果を反映します。売上増加、生産性向上、作業時間削減、雇用、地域波及などの数値は、根拠がなければ `[要確認]` としてください。

## `[要確認]` リスト

このテンプレートを使った後、未確認の要件、数値、添付資料、顧客本人の判断が必要な事項を `input/subsidy-fit.md` の `[要確認]` リストへ転記してください。
